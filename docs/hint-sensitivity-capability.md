# Hint Sensitivity vs SWE Capability

This note summarizes the current evidence for the hypothesis:

```text
Measuring how a model changes after adding hints can estimate model capability.
```

The current evidence does **not** validate the hypothesis as a standalone
capability estimator. Hint sensitivity is useful as a diagnostic signal: it can
show whether a model reacts to helpful hints, ignores irrelevant hints, or gets
pulled by misleading hints. It does not yet produce a stable ranking that
matches real SWE task success.

## Reader Map

This project evaluates models on CubeSandbox SWE tasks. A model must inspect a
real repository, modify code, and produce a patch. CubeSandbox then runs the
verifier and records whether the attempt passed.

Hint-Eval asks a narrower offline question. It takes an intermediate prefix
from a model or teacher trajectory, adds different hints, and measures how the
student model scores the target next action.

The evidence has two lines:

| Evidence line | What it measures | Why it matters |
| --- | --- | --- |
| Online SWE rollout | Whether a full patch passes the verifier. | This is the capability target. |
| Offline hint probe | How loss changes under no/correct/irrelevant/misleading hints. | This is the proposed cheap estimator. |

The hypothesis would need stable agreement between those two lines across
models, tasks, probes, and prefix sources. The current runs do not meet that
bar.

## Terms

| Term | Meaning |
| --- | --- |
| `task_id` | A CubeSandbox/SWE benchmark task id. It is a dataset instance id, not generated by the model. |
| Teacher | A model or trajectory source used to provide successful process traces and target actions. |
| Student | The model being evaluated. |
| Rollout / trajectory | One full attempt to solve a SWE task, including reads, commands, edits, and verification. |
| Prefix | A truncated trajectory state: "the model has reached this point; what should happen next?" |
| Probe | An offline scoring sample built from a prefix and candidate next actions. |
| Hint | Extra context added to a probe: neutral, correct/causal, irrelevant, or misleading. |
| Online success | Full task success. In this report, `ok=1`; `no_patch`, `failed`, and model-caused `error` count as 0. |

## Metrics

Loss values are target-action cross entropy values. Lower loss means the model
prefers the target action more strongly.

| Metric | Meaning | Preferred direction |
| --- | --- | --- |
| `L0` | Neutral/no-business-hint loss. | Lower |
| `L_plus` | Loss after the correct causal hint. | Lower |
| `L_irrelevant` | Loss after an irrelevant hint. | Close to `L0` |
| `L_misleading` | Loss after a misleading hint. | Context-dependent |
| `G_plus = L0 - L_plus` | Correct hint gain. Positive means the hint lowered target loss. | Higher, but not sufficient alone |
| `S_irrelevant = abs(L_irrelevant - L0)` | Disturbance from irrelevant hints. | Lower |
| `H_misleading = L_misleading - L0` | Change under misleading hints. | Diagnostic, sign needs interpretation |
| `B` | Existing composite burden score. | Lower |
| `Goodness = -B` | Higher-is-better orientation of `B`. | Higher |

`Goodness` is an offline diagnostic score, not an online benchmark score.

## Current Data Scope

The latest cross-check covers two teacher axes and three student models.

| Axis | Source | Coverage |
| --- | --- | --- |
| GPT-5.5 teacher | Previously collected successful teacher trajectories. | 71 success trajectories across 36 tasks. |
| Aliyun qwen3.7-max teacher | New teacher rollouts, keeping only verified successful trajectories. | 7 success trajectories across 6 tasks. |
| Students | Qwen3.6 27B, Affine, Qwen3 32B. | 50 tasks x 4 repeats = 200 online jobs per student. |

The Aliyun axis is a robustness check, not a replacement for the larger
GPT-5.5 teacher set. It is intentionally conservative: only verified successful
teacher trajectories are used.

The key mismatch is visible here:

| Ranking source | Ranking |
| --- | --- |

Both teacher axes produce the same offline Goodness ranking, but that ranking
does not match the online capability ranking. This means Goodness is measuring
some stable offline fit to the probe/teacher construction, but that fit is not
the same thing as SWE capability.

## Ablation Findings

The ablation replaces the composite `Goodness=-B` with simpler alternatives:

| Baseline | Question it asks |
| --- | --- |
| `-L0` | Is the model already close to the target without hints? |
| `-L_plus` | Is the model close to the target after the correct hint? |
| `G_plus` | Does the correct hint improve the target action? |
| `-S_irrelevant` | Is the model stable under irrelevant hints? |
| `-H_misleading` | Does the misleading-hint change contain useful signal? |
| `random` | How much noise can look competitive in this sample? |

Best alternative baselines by row:

| Teacher | Student | Best alternative | Spearman | Pairwise |
| --- | --- | --- | ---: | ---: |
| GPT-5.5 | Qwen3.6 27B | `-H_misleading` | 0.0239 | 0.5177 |
| GPT-5.5 | Affine | `-L0` | 0.0760 | 0.5743 |
| GPT-5.5 | Qwen3 32B | `-H_misleading` | 0.0411 | 0.5823 |
| Aliyun qwen3.7-max | Qwen3.6 27B | `random` | 0.0884 | 0.5823 |
| Aliyun qwen3.7-max | Affine | `-S_irrelevant` | 0.0715 | 0.5655 |
| Aliyun qwen3.7-max | Qwen3 32B | `-H_misleading` | 0.0521 | 0.6093 |

No single field wins consistently. The fact that `random` wins one small-sample
row is a warning that the Aliyun teacher success set is too small for strong
statistical claims.

## Prefix-Group Interpretation

Prefix groups ask whether the signal only works in specific states:

| Prefix group | Meaning |
| --- | --- |
| `all_teacher_success_prefix` | Prefixes from successful teacher trajectories. |
| `high_support_teacher_prefix` | Higher-support teacher prefixes. |
| `all_student_onpolicy_prefix` | Prefixes from the student model's own rollouts. |
| `student_success_prefix` | Prefixes from student rollouts that eventually passed. |
| `student_failure_prefix` | Prefixes from student rollouts that eventually failed. |

The prefix analysis supports a narrower conclusion: hint sensitivity is useful
for local behavior diagnosis. It can show that a model reacts differently in
teacher states versus its own failure states. The pattern does not yet combine
into a stable global capability estimate.

## Interpretation by Student

Qwen3.6 27B has the highest online success rate, `7/200`. Its `G_plus` is
positive under both teacher axes, so correct hints do move it toward the target
on average. Its Goodness remains low because neutral loss and irrelevant-hint
disturbance are high.

Affine has the best Goodness under both teacher axes, but only `2/200` online
success. This is the clearest counterexample to using Goodness as a direct
capability ranking.

Qwen3 32B has the lowest online success, `1/200`, and negative `G_plus` under
both teacher axes. That local signal agrees with weak online performance, but
the composite Goodness still ranks it above Qwen3.6 27B, so the global estimator
is not reliable.

## Reproduce

The cross-model report generator reads existing artifacts only. It does not call
model endpoints and does not run CubeSandbox:

```sh
.venv/bin/python scripts/analyze_hint_sensitivity_capability.py \
  --raw-task-limit 30 \
  --raw-probe-limit 30
```

Default outputs are intentionally under ignored `results/` paths:

```text
results/hint_eval_full/hint_sensitivity_capability_report.md
results/hint_eval_full/hint_sensitivity_capability_analysis.json
```

The Aliyun cross-teacher scoring pipeline is:

```sh
.venv/bin/python scripts/run_aliyun_teacher_cross_students.py \
  --cross-dir results/hint_eval_full/aliyun_teacher_cross_20260524T130410Z \
  --skip-scored
```

That script deploys/scoring students unless `--skip-deploy` is provided. It
expects the Targon and Aliyun/OpenAI-compatible endpoint configuration in
`.env`, and writes all run artifacts under ignored `results/`.

## Conclusion

Hint sensitivity is a useful diagnostic feature, but the current two-teacher,
three-student evidence does not support using it as a standalone capability
estimator. A stronger claim would require more teacher successes, more online
successes, task-level bootstrap confidence intervals, and consistent direction
after controlling for prefix source and task difficulty.
